# ADD Glossary（用語集）

## Lake（知識の湖）
- チケットに関する情報・仮説・不確実性・ユースケースなど  
  すべての“原材料”を集める場所。

## Rivers（フェーズ＝流路）
- Lakeから流れ出す価値フェーズ。
- 1〜2時間で終わる規模に分割する。

## Dams（ダム＝イテレーション）
- River上に置かれる小さな作業単位。
- 1ダム = 1〜2時間。
- **Probe（探索）** と **Build（実装）** が存在する。

## Probe Iteration（探索）
- 不確実性の高い部分を試す。
- プロトタイプ、軽い実験、PoCなど。

## Build Iteration（実装）
- テスト → 実装 → リファクタ の TDDで進めるイテレーション。

## Flow（流れ）
- Damに従って実際に作業が進んでいく過程。
- 何をやったかの記録（ログ）。

## Gold Sand（砂金）
- 最後に残る“学び・パターン・気づき”。
- 組織の知識として蓄積され続ける。

## ADD（Assist Driven Development）
- 人間主導 × AI補助で  
  小さなイテレーションを積み重ねる開発手法。

- 世界観は  
  **Lake → Rivers → Dams → Flow → Gold**  
  の水の循環で表現される。
