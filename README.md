# ADD (Assisted Driven Development)

**「AIをブラックボックスにするな、自らの知能を拡張する『スタンド』にせよ」**

ADDは、AIにコードを丸投げするのではなく、AIとの対話を通じてエンジニアの思考を資産化し、ホワイトボックスな開発を超高速に実現するための開発フレームワークです。

---

## 1. 開発哲学：なぜ ADD なのか？

現代のAI開発ツールの多くは、「いかに人間にコードを書かせないか」という結果（Output）に固執しています。しかし、その先にあるのは**誰も中身を理解できないブラックボックスの積み上げ**です。

ADD（Assisted Driven Development）は、あえて「プロセス」に規律を設けることで、以下の3つを達成します。

> ### 🧠 ホワイトボックスの追求
> 
> 
> エンジニアが100%理解し、説明責任を持てるコードだけをプロダクトに残す。AIは生成器ではなく、最強の「ペアプログラミング・パートナー」である。
> ### 📖 読む開発（Reading-Centric）
> 
> 
> ゼロからコードを書く苦労をAIに任せ、エンジニアは「設計」と「精査（レビュー）」、そして最後の「魂入れ（5点の磨き上げ）」にリソースを集中させる。
> ### 💎 思考の資産化
> 
> 
> 日々の試行錯誤は使い捨てず、`core/memory` に蓄積する。使えば使うほど、AIはあなたの癖やこだわりを学習し、世界に一人だけの「あなた専用の相棒」へと進化する。

---

## 2. 核心的な仕組み

ADDは、**「不変の資産（Core）」**と**「刹那の作業（Workspace）」**を完全に分離します。

### 📂 ディレクトリ構造の美学

* **`.add/core/`**: あなたの知能のコピー。Gitで管理され、プロジェクトを跨いで継承される「脳」です。
* **`.add/workspace/`**: 日々の泥臭い試行錯誤の場。タスクごとに隔離され、Gitからは除外されます。これにより、誰にも見られない自由な思考の爆発を許容します。

### 🛠️ 階層化されたイテレーション

1. **Phase**: 大きな設計図。全体像をマイルストーンに分割。
2. **Steps**: 現場の戦術。`[Red] -> [Green] -> [Refactor] -> [Manual] -> [CodeAnalysis]` という厳格なTDDサイクルを回し、品質を物理的に担保します。

---

## 3. 強力なカスタムコマンド

ADD環境にインストールされたAIエージェントは、以下の日本語コマンドであなたの思考をサポートします。

* **「ADDで開始して」**: タスク専用の隔離された作業部屋を一瞬で準備します。
* **「設計（Phase）して」**: 情報を分析し、下位モジュールから積み上げるボトムアップな計画を立てます。
* **「しおりを挟んで」**: 別の急な作業が入っても、現在の思考状態を完璧にダンプして中断できます。
* **「しおりから再開して」**: 数日後でも、AIが当時のコンテキストを復旧し、即座に作業に戻れます。
* **「昇華（Learn）して」**: 今回のタスクで得た「あなたのこだわり」をCoreに書き戻し、AIを成長させます。

---

## 4. ワークフロー：95点から100点へ

AIが弾き出す95点の成果物に対し、エンジニアが最後の5点を直接手で修正する。この「手触り感」こそがADDの真髄です。
AIに指示を繰り返す（プロンプト・エンジニアリング）よりも、自分で書いたほうが早い「ラストワンマイル」を大切にすることで、圧倒的な開発速度と納得感を両立します。

---

## 5. 導入方法

1. プロジェクトルートで `setup_add.sh` を実行。
2. `.add/core/memory/user-preference.md` にあなたのこだわりを記述。
3. AIエージェントに **「ADDで開始して」** と告げる。

---

**「エンジニアが支配権を握ったまま、AIの速度で山を登る。」**
それが、Assisted Driven Development です。
