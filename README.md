# Assist Driven Development (ADD)

**Assist Driven Development（ADD）** は、  
人間を主役に据え、AIを“補助（Assist）”として活用するために設計された  
新しい開発手法です。

ADD は、以下のような AI 時代の開発の“痛み”を解決することを目的としています：

- 仕様書をAIに渡しても誤解される
- 大量コード生成によるレビュー地獄
- バイブコーディング化による保守性の崩壊
- モデルのコンテキスト限界やセッション切れ
- AIが停止すると人間が引き継げない構造的な脆さ

ADD はこれらを  
**プロセス設計と世界観の再構築** によって解決します。

---

# 🌊 ADDとは

ADD は開発を、自然界の「水の流れ」を使って表現します。

- **Lake（知識の湖）**  
- **Rivers（価値単位のフェーズ）**  
- **Dams（小さなイテレーション）**  
- **Flow（作業の進行）**  
- **Gold Sand（最後に得られる学び）**

これにより、大きな課題を自然に分解し、  
人間とAIが安全に協働できるプロセスが成立します。

# ⭐ ADDの7原則（Principles）

ADDは以下の7原則を軸に設計されています：

1. **Human-First Principle（人間主導）**  
2. **Small Phases Principle（極小フェーズ）**  
3. **Localized AI Assistance Principle（局所AI活用）**  
4. **Rapid Realignment Principle（高速再アラインメント）**  
5. **Human Recoverability Principle（人間復旧可能性）**  
6. **Test-First Execution Principle（テスト先行/TDD）**  
7. **Continuous Refinement Principle（継続的洗練）**

詳細は `docs/add_overview.md` を参照してください。

# 🔧 ADDは何を解決するか？

- AI丸投げで破綻する現代の開発フロー  
- AI主導によるブラックボックス化  
- コンテキスト切れ・不安定性への脆弱性  
- 大規模生成のレビュー不可性  
- 人間の判断・設計力の喪失  

ADD はこれらに対し、

- **人間が主役でい続けられる構造**  
- **AIを“小さく・安全に”使う設計**  
- **フェーズの再設計による破綻防止**  
- **ナレッジ蓄積（Lake / Gold）**  
- **人間だけで継続できる冗長性**

を提供します。
